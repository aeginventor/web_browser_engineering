## 🛠️ 개발 일지: 브라우저의 진화 과정

지금까지 구현한 브라우저가 어떻게 발전해왔는지 기록합니다.

### Chapter 1: 웹페이지 다운로드

> 웹 서버와 연결 해 단순히 웹페이지를 다운로드해 콘솔 창에 텍스트를 표시하는 브라우저

- **구현 내용:** `socket`과 `ssl` 라이브러리를 사용해 웹 서버와 TCP 연결을 맺고, HTTP GET 요청을 보내 응답을 받아왔습니다.
- **결과:** 터미널 창에 HTML 코드가 문자열 그대로 출력되는 형태였습니다. HTML 태그(`<div...`)와 본문 내용이 섞여 있어 읽기 힘들었습니다.

### Chapter 2: 화면에 그리기

> 윈도우 창에 표시되고 스크롤을 내릴 수 있는 브라우저

- **구현 내용:** `tkinter`를 도입하여 실제 윈도우 창을 띄웠습니다. HTML 태그를 제거하는 필터링을 거친 후, `Canvas`에 글자들을 하나씩 그렸습니다.
- **발전:** 스크롤 기능을 구현하여 긴 웹페이지도 볼 수 있게 되었고, 화면 밖의 글자는 그리지 않는 최적화를 통해 성능을 확보했습니다. 하지만 모든 글자의 크기가 같을 때만 유효한 구현이고 줄바꿈이 단순한 상태입니다.

### Chapter 3: 텍스트 포맷팅하기

> 일반적인 영어 문장을 읽을 수 있는 브라우저

- **구현 내용:** 텍스트를 측정하고 한 단어씩 처리하여 영문도 읽을 수 있게 되었으며, `<b>`(굵게), `<i>`(이탤릭) 같은 태그를 인식해 스타일을 입힐 수 있게 되었습니다. 레이아웃 엔진을 분리하여 설계했습니다.
- **발전:** 폰트마다 다른 너비와 높이를 계산하여 다양한 크기의 텍스트가 섞여있어도 일관된 줄바꿈을 할 수 있게 됐고, 텍스트 측정으로 인한 성능 저하를 폰트 캐싱으로 다소 해결했습니다.
